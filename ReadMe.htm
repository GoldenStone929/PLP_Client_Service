Using the Java Client SDK for Pipeline Pilot Server

The Java Client SDK is a set of pre-built Java classes that allows you to create a custom SOAP client for the Pipeline Pilot server. This set of classes expedites the process of creating a client by abstracting and hiding much of the complexity involved in creating SOAP clients. It also provides an object-oriented interface that you may find familiar.

Building the Example

The root folder contains the following aids for Windows development:

  • compile.bat: An example batch file with a command line to build the source code.
  • run.bat: A file that you can use to execute com.scitegic.proxy.examples.NewTree.

Tip: The example requires jar files to build and execute. These files are located in "<install>/apps/scitegic/clientsdkjava/bin", where <install> is the directory where the Pipeline Pilot server is installed.

Running the Example

This Java example runs one of the example Web services protocols. It is intended to act as a starting point to help you write Java client scripts to access your own Web services protocols on the server.

To run the example on Windows:

  1. Open a command shell and change directory to the root folder of this folder.
  2. Run "compile.bat" in the shell.
  3. Run the example with the "run.bat" batch file.
  4. The syntax is: run.bat <server_URL> <username> <password>.
    • For HTTP, use "http://<servername:port>" for <server_URL>.
    • For SSL, use "https://<servername:port> for <server_URL>.
    • <servername> is the name of the Pipeline Pilot server on your network, and <port> is the HTTP or HTTPS port for which your server accepts SOAP calls.

Tip: If you copy this example to another location, you also need to copy the jar files and edit the paths in the batch file to the jars and "jdk/bin" directory.
